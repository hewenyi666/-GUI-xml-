该工具的用途为:生成xml格式的测试用例(数据),使用 python3+Tkinter 完成的一款GUI测试工具

开发背景:以往用的用例是xml格式的,生成测试用例的话,需要到xml文件内部修改数据,来回CTRL+CV,特别容易出错,而且效率很低.所以开发了这款测试工具,用来提升测试效率

文件说明:

final testcase文件夹里面的result_case.xml文件,即为最终生成的测试用例

temporary data文件夹不用管,里面放的是模板

xml tool.py 是这款GUI工具的源码(python3版本)

use tool.bat 运行文件的快捷方式,需要电脑默认为python3版本,不是的话,稍作修改文件中内容即可!

使用方式:
1.提前建好想要input或者output的文件,有的输入框后面有选择文件选项,点击选择文件的按钮,然后选择想要input或者output文件的路径,
即可将已选择文件的路径填入输入框中;有单选框的选项,直接点击对应项即可(目前提供可选项不多,可后续添加).

2.待数据都填写完毕后,点击"开始添加"按钮,可查看result_case.xml文件,会发现用例已成功添加进去.若想结束添加用例,则点击"
结束添加"按钮,此时会往文件中添加</testcases>,此时xml文件结构完整可用.

3.若点击过"结束添加"按钮,但是还想继续添加用例,则点击最下方的"继续添加"按钮(此时会去除文件中的</testcases>),然后继续正常添加
用例即可,重复第二步.

备注:在打开这个工具后,result_case.xml文件中的用例会清空,只保留信息头,所以需要提前做好保存数据工作!

优点:
1.通过制作该工具,可以更加熟悉Tkinter的常用组件,如Label,Entry,LabelFrame,常用布局方式等,如grid,pack.
2.如果后续需要使用xml文件生成测试用例的话,可以参考这个项目,生成类似的GUI测试工具,提升测试效率

存在的不足:其中有一部分需要实现个需求,将1个字符串中的14个字符串,分别替换成另一个字符串(大致是这个意思),例如,text1中的1个字符换成新的字符,取得结果为text2,然后继续替换text2中的某个字符串为一个新的字符串得到text3...一直这样,直到全部替换完成.目前使用的是多次replace,没实现更简便的方法,等想出来后,后续优化吧.
